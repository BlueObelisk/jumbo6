Examples

These exampes are to test the code, not the data or the schema.

They comprise:
  • complex: one or a very few valid and static medium/large examples to multiple namespaces, performance, etc..
  • dict: one or a very few valid and static dictionaries to test CMLDictionary and the linkage of dictRef and other namespaceRef tools.
  • units: one or a very few valid and static unitLists to test CMLUnit and the linkage of units attributes.
  • xsd: small examples specifically for each schema element. This is to test that CMLNodeFactory works for each element. If it can be guaranteed, then these examples might go.
